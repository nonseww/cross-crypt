# User Cases

### UC-1: Создание игровой комнаты и подключение друзей

| Элемент прецедента | Описание |
| :--- | :--- |
| **Название** | Создание игровой комнаты и подключение друзей |
| **Участники (Акторы)** | Игрок (Инициатор), Друзья игрока, Система |
| **Предусловия** | Игрок и его друзья авторизованы в Discord и запустили веб-игру CrossCrypt. |
| **Триггер** | Игрок нажимает кнопку «Создать игровую комнату» в главном меню игры. |
| **Основной сценарий** | 1. Система создает новую игровую сессию и генерирует уникальный код комнаты.<br>2. Система автоматически добавляет Игрока в созданную комнату в роли создателя лобби.<br>3. Система отображает интерфейс лобби, код комнаты и кнопку «Пригласить друзей».<br>4. Игрок копирует код или ссылку-приглашение и отправляет её друзьям в Discord.<br>5. Друзья игрока вводят код (или переходят по ссылке) и подключаются к комнате.<br>6. Система обновляет список участников в лобби в реальном времени для всех игроков. |
| **Расширения (Альтернативные сценарии)** | **1а. Сбой интеграции с Discord (ошибка авторизации):**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система выводит сообщение об ошибке подключения к Discord.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Система предлагает повторить попытку авторизации.<br>**5а. К комнате пытается подключиться четвертый игрок:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система блокирует вход для данного пользователя.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Система выводит ему уведомление «Комната уже заполнена (максимум 3 игрока)».<br>**5б. Друг ввел несуществующий или устаревший код комнаты:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система выдает ошибку «Комната не найдена».<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Система предлагает проверить корректность введенного кода. |

### UC-2: Выбор и получение игровой роли

| Элемент прецедента | Описание |
| :--- | :--- |
| **Название** | Выбор и получение игровой роли |
| **Участники (Акторы)** | Игрок (Инициатор), Система |
| **Предусловия** | Игрок находится в созданной игровой комнате. Сессия еще не запущена. |
| **Триггер** | Игрок нажимает кнопку выбора роли или «Готов» в интерфейсе лобби. |
| **Основной сценарий** | 1. Система отображает доступные роли: Хакер, Аналитик, Криптограф.<br>2. Игрок выбирает одну из свободных ролей.<br>3. Система закрепляет роль за игроком и обновляет статус в лобби для всех участников.<br>4. Система проверяет готовность всех трех участников комнаты. |
| **Расширения (Альтернативные сценарии)** | **2а. Выбранная роль уже занята другим игроком:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система выводит уведомление «Роль уже занята».<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Система предлагает выбрать другую свободную роль.<br>**4а. Все три роли успешно распределены:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система активирует кнопку запуска матча для создателя комнаты. |

### UC-3: Получение информации для решения кроссворда

| Элемент прецедента | Описание |
| :--- | :--- |
| **Название** | Получение информации для решения кроссворда |
| **Участники (Акторы)** | Аналитик (Инициатор), Криптограф, Система |
| **Предусловия** | Игровая сессия запущена. Роль Аналитика успешно присвоена игроку. |
| **Триггер** | Начало раунда или выбор конкретного номера слова на игровом экране. |
| **Основной сценарий** | 1. Система отображает Аналитику список текстовых вопросов и правильные ответы.<br>2. Аналитик выбирает нужный вопрос для разгадки.<br>3. Аналитик сопоставляет данные и устно передает буквы правильного ответа Криптографу. |

| **Расширения (Альтернативные сценарии)** | **1а. Произошла динамическая смена ключей/вопросов:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система мгновенно обновляет список доступных вопросов на экране Аналитика.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Аналитик продолжает сценарий с шага 2 с новыми данными. |

### UC-4: Шифрование букв и передача символов Хакеру

| Элемент прецедента | Описание |
| :--- | :--- |
| **Название** | Шифрование букв и передача символов Хакеру |
| **Участники (Акторы)** | Криптограф (Инициатор), Хакер, Система |
| **Предусловия** | Игровая сессия запущена. Криптограф получил буквы от Аналитика. |
| **Триггер** | Появление таблицы шифрования на экране или получение новой буквы для перевода. |
| **Основной сценарий** | 1. Система выводит на экран Криптографа таблицу соответствия «буква → символ».<br>2. Криптограф находит в таблице буквы, переданные Аналитиком.<br>3. Криптограф определяет зашифрованные символы.<br>4. Криптограф устно (через голосовой канал) диктует полученные символы Хакеру. |
| **Расширения (Альтернативные сценарии)** | **1а. Сработал таймер динамической смены ключей шифрования:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система обновляет значения в таблице соответствия «буква → символ».<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Система подает звуковой/визуальный сигнал Криптографу об изменении ключа.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. Криптограф выполняет перевод заново по обновленной таблице. |

### UC-5: Заполнение кроссворда и контроль игрового таймера

| Элемент прецедента | Описание |
| :--- | :--- |
| **Название** | Заполнение кроссворда и контроль игрового таймера |
| **Участники (Акторы)** | Хакер (Инициатор), Система |
| **Предусловия** | Игровая сессия активна. Хакер получил символы от Криптографа. |
| **Триггер** | Хакер кликает на ячейку кроссворда для ввода символа. |
| **Основной сценарий** | 1. Система активирует выбранную Хакером ячейку сетки.<br>2. Хакер вводит полученный символ с клавиатуры.<br>3. Система мгновенно отображает введенный символ в ячейке и синхронизирует прогресс.<br>4. Система проверяет финальное состояние кроссворда и завершает игру при успехе или истечении времени. |
| **Расширения (Альтернативные сценарии)** | **2а. Хакер совершил ошибку при вводе:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Хакер выделяет ячейку и редактирует/удаляет символ.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Система обновляет отображение на экране.<br>**3а. Введенный символ некорректен (ошибка в решении):**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система фиксирует ошибку, применяет штраф (отнимает время на общем таймере).<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Система визуально сигнализирует Хакеру об ошибке.<br>**4а. Кроссворд полностью и верно заполнен до завершения таймера:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система останавливает таймер, фиксирует победу команды и завершает сессию.<br>**4б. Общий игровой таймер дошел до 0:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система блокирует ввод символов, объявляет о проигрыше (терминал заблокирован) и завершает матч. |

### UC-6: Модерация игрового контента в словаре

| Элемент прецедента | Описание |
| :--- | :--- |
| **Название** | Модерация игрового контента в словаре |
| **Участники (Акторы)** | Администратор (Инициатор), Система |
| **Предусловия** | Администратор авторизован в панели управления / админ-интерфейсе игры CrossCrypt. |
| **Триггер** | Администратор переходит в раздел «Управление словарем». |
| **Основной сценарий** | 1. Система отображает текущий список слов, связанных с ними вопросов и ответов.<br>2. Администратор нажимает кнопку «Добавить слово» (или выбирает «Редактировать» у существующего).<br>3. Администратор вводит/изменяет текст вопроса и правильный ответ.<br>4. Администратор сохраняет изменения.<br>5. Система валидирует данные и обновляет базу данных игрового контента. |
| **Расширения (Альтернативные сценарии)** | **5а. Введенные администратором данные не прошли валидацию (например, пустое поле):**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Система подсвечивает ошибочное поле красным цветом.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Система блокирует сохранение до исправления ошибки. |
